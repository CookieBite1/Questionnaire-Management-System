# Βήματα Εκτέλεσης
```bash
git clone <repository-url>
cd <project-folder>
```

Εκκίνηση με Docker:
```bash
docker-compose up --build
```
Φόρτωση αρχικών δεδομένων (αν χρειαστεί, όμως υπάρχει volume στο docker):
```bash
docker exec -it flask_app python scripts/load_data.py
```
Πρόσβαση:
```bash
http://localhost:5001/
```

# Πίνακας Ρίσκων Υλοποίησης
Ο πίνακας ρίσκων περιλαμβάνει πιθανούς κινδύνους κατά την υλοποίηση του συστήματος, κατανεμημένους με βάση την πιθανότητα εμφάνισης και τον βαθμό επίδρασής τους. Για κάθε ρίσκο προτείνονται μέθοδοι αντιμετώπισης, όπως χρήση docker-compose, αποθήκευση αντιγράφων ασφαλείας και επικυρώσεις πεδίων εισόδου. Ο πίνακας αυτός βοηθά στην πρόβλεψη και πρόληψη προβλημάτων κατά την ανάπτυξη και χρήση της εφαρμογής.

| Κωδικός | Περιγραφή Ρίσκου                            | Πιθανότητα | Επίδραση | Αντιμετώπιση                                  |
| ------- | ------------------------------------------- | ---------- | -------- | --------------------------------------------- |
| R1      | Αποτυχία σύνδεσης με MongoDB                | Υψηλή      | Υψηλή    | Έλεγχος docker-compose                        |
| R2      | Καθυστέρηση στην είσοδο των φοιτητών        | Μεσαία     | Μεσαία   | Δημιουργία script για προκαθορισμένα accounts |
| R3      | Χρήστες απαντούν λανθασμένα (κακή εμπειρία) | Μεσαία     | Χαμηλή   | Φιλικό UI/UX και επικυρώσεις                  |
| R4      | Απώλεια δεδομένων                           | Χαμηλή     | Υψηλή    | Backup/Restore και logging                    |
| R5      | Παραβίαση ασφάλειας (login/admin access)    | Μεσαία     | Υψηλή    | Hash κωδικών, session controls                |

## Περιγραφή
Το σύστημα ξεκινά με την είσοδο στο σύστημα είτε από Φοιτητή, είτε από Διαχειριστή, ενώ ο Απλός Χρήστης μπορεί να πλοηγηθεί και χωρίς σύνδεση. Ο φοιτητής έχει τη δυνατότητα να δημιουργεί ερωτηματολόγια, να προβάλλει και να επεξεργάζεται τα δικά του, καθώς και να παρακολουθεί τις απαντήσεις που λαμβάνει. Οι χρήστες (φοιτητές ή απλοί) μπορούν να προβάλουν ερωτηματολόγια και να απαντούν σε αυτά μέσω μοναδικού URL ή λίστας διαθέσιμων ερωτηματολογίων. Ο διαχειριστής μπορεί να διαχειρίζεται φοιτητές (προσθήκη/διαγραφή) και να παρακολουθεί τη δραστηριότητα. Όλες οι ενέργειες γίνονται σε σύνδεση με τη βάση δεδομένων MongoDB, η οποία αποθηκεύει πληροφορίες φοιτητών, ερωτηματολογίων και απαντήσεων.
🧑‍💻 Τρόπος Χρήσης Συστήματος
🎓 Φοιτητής:
- Σύνδεση μέσω /student/login
- Δημιουργία / Προβολή / Τροποποίηση ερωτηματολογίων
- Προβολή απαντήσεων στα ερωτηματολόγιά του

🔐 Διαχειριστής:
- Αυτόματη ύπαρξη χρήστη: username: admin, password: admin123
- Διαχείριση φοιτητών (προσθήκη/διαγραφή)
- Προβολή όλων των ερωτηματολογίων

👤 Απλός Χρήστης:
- Πρόσβαση στα διαθέσιμα ερωτηματολόγια χωρίς login
- Αναζήτηση / Ταξινόμηση
- Απάντηση σε ερωτηματολόγια
